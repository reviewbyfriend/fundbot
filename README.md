# FundBot Office Production

บอทเก็บเงินกองกลางสำนักงานใน LINE กลุ่ม คล้ายขุนทอง แต่ปรับสำหรับงานกองกลางรายเดือนที่แต่ละคนยอดไม่เท่ากัน และมีรายจ่ายผันแปรทุกเดือน

## ทำอะไรได้ในเวอร์ชันนี้

- เชิญ LINE OA Bot เข้า LINE กลุ่มเดิมได้
- เปิดรอบเดือน เช่น `เปิดรอบ กรกฎาคม 2569 ยกมา 17813.50`
- สมาชิกแต่ละคนยอดไม่เท่ากัน
- สมาชิกลงทะเบียนผูกชื่อกับ LINE ID เช่น `ลงทะเบียน รักษิน`
- สมาชิกพิมพ์ `ชำระเงิน` แล้วได้ปุ่มเปิด QR PromptPay ตามยอดค้างของตัวเอง
- โอนแล้วส่งสลิปในกลุ่ม และพิมพ์ `จ่ายแล้ว 500` เพื่อให้ระบบตัดยอด
- แอดมินบันทึกแทนได้ เช่น `รับเงิน รักษิน 500`
- เพิ่มรายจ่าย เช่น `รายจ่าย ค่าน้ำ 1816`
- สรุปยอด/ทวงเงิน/สร้างรายงาน Excel
- พร้อม Deploy Railway ด้วย Dockerfile

> หมายเหตุสำคัญ: ระบบ “รับรูปสลิป” แล้ว แต่ยังไม่ตัดยอดจากสลิปอัตโนมัติ 100% เพราะการอ่านและตรวจสลิปจริงควรต่อ OCR/Slip Verify API เพื่อกันสลิปปลอม/ยอดผิด เวอร์ชันนี้ให้พิมพ์ยืนยันยอดหลังส่งสลิปก่อน ใช้ได้จริงในสำนักงานแบบควบคุมโดยแอดมิน

1. Railway → New Project

4. รอ Deploy จนขึ้น Online

## 3) เพิ่ม PostgreSQL บน Railway

ใน Project เดิม กด

```text
+ Add → Database → PostgreSQL
```

Railway จะสร้างตัวแปร `DATABASE_URL` ให้เอง

---

## 4) ตั้ง Variables ใน Railway

ไปที่ Service `web` → Variables แล้วเพิ่ม:

```env
APP_BASE_URL=https://ชื่อโปรเจกต์.up.railway.app
LINE_CHANNEL_ACCESS_TOKEN=เอาจาก LINE Developers
LINE_CHANNEL_SECRET=เอาจาก LINE Developers
PROMPTPAY_ID=เบอร์พร้อมเพย์หรือเลขบัตรประชาชน
ADMIN_USER_IDS=LINE userId ของแอดมิน คั่นด้วย comma
OFFICE_NAME=สำนักงานอัยการพิเศษฝ่ายคดีล้มละลาย ๑
FUND_NAME=เงินกองกลางสำนักงาน
```

`DATABASE_URL` ปกติ Railway จะใส่ให้เมื่อเชื่อม PostgreSQL แล้ว

### ช่วงทดสอบ
ถ้ายังไม่รู้ `ADMIN_USER_IDS` ให้เว้นว่างไว้ก่อน ระบบจะอนุญาตให้ทุกคนใช้คำสั่งแอดมินได้ชั่วคราว จากนั้นค่อยใส่จริงก่อนใช้งานในกลุ่มสำนักงาน

## 7) Flow ใช้งานจริง

1. แอดมินเพิ่มสมาชิกและยอดประจำตัว หรือใช้คำสั่งนำเข้าไฟล์ตัวอย่าง
2. แอดมินเปิดรอบเดือน
3. สมาชิกลงทะเบียนชื่อครั้งแรก
4. สมาชิกพิมพ์ `ชำระเงิน`
5. ระบบส่งปุ่มเปิด QR PromptPay
6. สมาชิกโอนในแอปธนาคาร
7. สมาชิกส่งสลิปในกลุ่ม แล้วพิมพ์ `จ่ายแล้ว 500`
8. ระบบขึ้นสถานะจ่ายแล้ว
9. แอดมินพิมพ์ `สรุป` หรือ `ทวงเงิน`
10. สิ้นเดือนพิมพ์ `รายงาน` เพื่อโหลด Excel

## 8) ต่อ OCR อ่านสลิปใน Phase ถัดไป

จุดต่ออยู่ที่ `app/main.py` ส่วน `msg_type == "image"`

สามารถเพิ่มได้ 2 วิธี:

1. OCR อ่านข้อความจากสลิป เช่น Gemini Vision / Google Vision / OCR.space
2. Slip Verify API ตรวจ ref/ยอด/บัญชีปลายทางจริง

แนะนำให้ใช้ Slip Verify API สำหรับงานเงินจริง เพราะปลอดภัยกว่า OCR อย่างเดียว


## OCR สลิปอัตโนมัติ

เวอร์ชันนี้เพิ่ม OCR สลิปแล้ว: สมาชิกส่งรูปสลิปใน LINE กลุ่ม → ระบบดาวน์โหลดรูป → ส่ง OCR → อ่านยอดเงิน/เลขอ้างอิง → ตัดยอดคนที่ลงทะเบียนไว้กับ LINE ID อัตโนมัติ

Variables เพิ่มใน Railway:

```text
OCR_SPACE_API_KEY=ใส่คีย์ OCR.Space
SLIP_RECEIVER_KEYWORDS=ชื่อบัญชีผู้รับ,เลขท้ายบัญชี,ชื่อกองกลาง
```

ถ้าไม่ใส่ `OCR_SPACE_API_KEY` ระบบจะใช้ demo key ของ OCR.Space ซึ่งอาจติด limit ได้ง่าย เหมาะแค่ทดสอบสั้นๆ

ข้อควรรู้:
- ต้องให้สมาชิกพิมพ์ `ลงทะเบียน ชื่อ` ก่อนอย่างน้อย 1 ครั้ง เพื่อผูก LINE ID กับชื่อในระบบ
- ระบบกันสลิปซ้ำด้วยเลขอ้างอิง ถ้า OCR อ่านเลขอ้างอิงเจอ
- ถ้า OCR อ่านยอดไม่ชัด ระบบจะไม่ตัดยอด และให้พิมพ์ยืนยัน `จ่ายแล้ว 500` แทน
- OCR ไม่ใช่ Slip Verify จากธนาคาร 100% ถ้าจะใช้กับเงินจำนวนมาก ควรต่อบริการตรวจสอบสลิปจริงเพิ่มเติม
